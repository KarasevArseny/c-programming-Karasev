# Версионирование динамических библиотек в Linux

Демонстрационный проект для изучения механизма версионирования shared libraries в Linux с использованием soname.

## Что такое версионирование библиотек?

Версионирование позволяет:
- Одновременно использовать разные версии библиотеки
- Обеспечивать обратную совместимость
- Правильно обновлять библиотеки в системе

## Компоненты версионирования

| Компонент | Пример | Описание |
|-----------|--------|----------|
| **Real name** | `libcalc.so.1.0.0` | Реальный файл с полной версией |
| **Soname** | `libcalc.so.1` | Символическая ссылка на real name, содержит только мажорную версию |
| **Linker name** | `libcalc.so` | Символическая ссылка для компиляции |

## Структура после сборки
libcalc.so.1.0.0 # Реальный файл библиотеки
libcalc.so.1 # Ссылка -> libcalc.so.1.0.0 (soname)
libcalc.so # Ссылка -> libcalc.so.1 (linker name)

## Сборка проекта

```bash
# Сборка всего
make

# Только статическая библиотека
make static

# Только динамическая с версионированием
make shared

# Создание символических ссылок
make symlinks

# Проверка версионирования
make check-version

# Информация о собранных файлах
make info

# Запуск тестов
make test          # статическая версия
make test-shared   # динамическая версия

# Полная проверка
make check

# Очистка
make clean
